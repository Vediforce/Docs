# Документация API Sidecraft

Данная документация описывает протокол взаимодействия между основным Telegram-ботом и подключенными игровыми серверами (комьюнити-проектами) через TCP-соединение `sidecraft`.

---

## 1. Общие принципы

Все сообщения между ботом и сервером передаются по TCP в формате JSON. Каждый пакет состоит из двух основных полей:

*   **`command`** *(string)* — строковый идентификатор события или действия.
*   **`payload`** *(dict)* — словарь с данными запроса или ответа.

---

## 2. Системные кнопки (Специальные действия в Telegram)

Бот умеет перехватывать определенные `callback_data` в клавиатурах, которые присылает сервер, и превращать их в нативные функции Telegram (например, ожидание ввода текста или управление подпиской).

Чтобы использовать их, сервер должен отправить клавиатуру (через `set_message`), где у нужной кнопки в поле `callback_data` указана одна из **зарезервированных команд**:

| `callback_data` от сервера | Что сделает бот при нажатии пользователем |
| :--- | :--- |
| `"handle_message"` | Переведет пользователя в режим ожидания текста. Пришлет ему сообщение: *"Отправьте сообщение..."*. Когда пользователь напишет текст, бот перешлет его на сервер (команда `handle_message`). |
| `"subscribe_notify"` | Подпишет пользователя на рассылки от вашего сервера и выдаст уведомление об успехе. Серверу ничего обрабатывать не нужно. |
| `"unsubscribe_notify"` | Отпишет пользователя от рассылок вашего сервера. |

**Пример кнопки для ввода текста:**
```json
{
  "text": "✍️ Написать сообщение чат", 
  "callback_data": "handle_message"
}
```

---

## 3. Запросы: Сервер ➔ Бот
*Команды, которые игровой сервер отправляет боту для управления интерфейсом или получения данных.*

### `set_message`
Обновляет текущее сообщение пользователя в Telegram (меняет текст и клавиатуру).

**Payload:**
```json
{
  "text": "Вы вошли в таверну. Что будете делать?",
  "keyboard": {
    "rows": [
      {
        "buttons":[
          {"text": "Поговорить с барменом", "callback_data": "talk_barmen"},
          {"text": "Написать на доске объявлений", "callback_data": "handle_message"}
        ]
      }
    ]
  },
  "original_payload": {
    "user_id": 123456,
    "message_id": 987
  }
}
```
*Примечание: Обычные callback'и (как `talk_barmen`) бот автоматически обернет в нужный формат и вернет вам при нажатии. Системные (как `handle_message`) бот обработает сам.*

### `get_subscribers`
Запрашивает список ID пользователей, которые подписаны на уведомления вашего сервера.

**Payload (запрос):**
```json
{
  "user_ids_to_check":[123, 456, 789] // (Опционально) Фильтр: проверить только этих юзеров
}
```
**Ответ от бота (придет команда `subscribers_list`):**
```json
{
  "users": [123, 789]
}
```

### `send_notification`
Массовая или одиночная отправка уведомлений подписанным пользователям (рассылка).

**Payload:**
```json
{
  "users": [123, 456], 
  "text": "Внимание! На сервер добавлен новый босс!",
  "keyboard": { ... } // (Опционально) Клавиатура к сообщению
}
```
**Лимиты (Rate Limits):**
*   **Одиночная отправка (1 юзер):** не чаще 1 раза в 60 секунд.
*   **Массовая отправка (>1 юзера):** не чаще 1 раза в 600 секунд (10 минут).
*   *Важно:* Если у запрошенных пользователей нет подписки, кулдаун не засчитывается.

## 4. События: Бот ➔ Сервер
*Команды, которые бот присылает на сервер в ответ на действия пользователя.*

### `game_enter`
Пользователь нажал кнопку входа на ваш сервер в главном меню бота.

**Payload:**
```json
{
  "user_id": 123456,
  "message_id": 987,
  "cls_name": "11б",           // Класс пользователя
  "privileges": ["admin", "vip"]   // Привилегии/Предметы
}
```

### `game_action`
Пользователь нажал на любую вашу кастомную кнопку (которую вы отправили через `set_message`).

**Payload:**
```json
{
  "user_id": 123456,
  "message_id": 987,
  "action": "talk_barmen"  // То, что вы указывали в callback_data
}
```

### `handle_message`
Срабатывает после того, как пользователь нажал системную кнопку `"handle_message"` и отправил боту текстовое сообщение.

**Payload:**
```json
{
  "user_id": 123456,
  "message_id": 987,
  "text": "Привет! Продам меч за 100 золотых."
}
```
*После получения этого ивента сервер обычно отвечает командой `set_message`, чтобы вернуть пользователя обратно в игровое меню.*
